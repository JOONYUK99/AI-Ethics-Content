[국가 인공지능 윤리기준(수업용 요약, 명칭 고정)]
- 프라이버시 보호: 개인정보를 함부로 모으거나 공개하지 않기.
- 연대성: 함께 안전하게 쓰는 규칙을 만들고 서로 돕기.
- 데이터 관리: 자료(데이터)를 정리하고, 출처/허락/목적을 확인하기.
- 침해 금지: 남의 권리(저작권/초상권/개인정보)를 해치지 않기.
- 안전성: 위험한 결과(유출/오해/피해)가 나지 않게 미리 점검하기.

[초등 5~6 연계 포인트(예시)]
- 실과: 정보 활용, 문제 해결, 안전한 사용 습관
- 도덕: 배려, 책임, 규칙, 공정

[수업 설계 힌트(교사용)]
- 학생 답은 ‘왜 그렇게 생각하나요?’로 근거를 끌어내기
- 교사 기준(출처/허락/목적/공개범위/대안)을 다음 행동에 반영하기
- 법 조항 단정 대신 “약관/규정/상황 확인 필요” 표현 권장

========================
[사례01] 저작권: 로고 제작 대회
- 주제: 저작권 / 침해 금지 / 데이터 관리
- 상황: 학급 로고 제작 대회가 열렸다. 어떤 친구가 유명 캐릭터 느낌이 나는 로고를 만들자고 한다.
- 쟁점: 유명 캐릭터/로고와 비슷하면 문제가 될 수 있다. 우리 반만의 새 아이디어가 필요하다.
- 선택 A: 비슷한 요소를 빼고, 우리 반만의 상징(색/도형/동물)을 새로 설계한다.
- 선택 B: 유명 캐릭터와 매우 비슷하게 만들고, “학교 안에서만 쓰면 괜찮다”고 생각한다.
- 확인 질문: 어디까지 비슷하면 ‘따라한 것’이 될까? 출처/허락 확인이 필요한가?
- 관련 윤리기준: 데이터 관리, 침해 금지, 안전성

========================
[사례02] 개인정보: 사진이 들어간 발표자료
- 주제: 개인정보 / 프라이버시 보호 / 안전성
- 상황: 발표자료에 친구 사진과 이름을 넣으면 더 이해하기 쉽다고 한다. 하지만 온라인 공개 가능성도 있다.
- 쟁점: 공개범위가 넓어지면 개인정보가 퍼질 수 있다. 허락과 대체 방법이 필요하다.
- 선택 A: 얼굴/이름을 빼고, 그림/아이콘으로 바꾼다. 필요한 경우에만 허락을 받는다.
- 선택 B: 발표가 재미있으니 사진/이름을 그대로 넣고 공유한다.
- 확인 질문: 누가 볼 수 있나(반/학교/온라인)? 허락을 어떻게 받나?
- 관련 윤리기준: 프라이버시 보호, 안전성, 침해 금지

========================
[사례03] 추천 알고리즘: 한쪽 정보만 보게 됨
- 주제: 추천 알고리즘 / 데이터 관리 / 연대성
- 상황: 같은 주제인데 친구마다 추천 영상이 다르다. 어떤 친구는 한쪽 주장만 계속 보게 된다.
- 쟁점: 한쪽으로 치우치면 판단이 어려워진다. 반대 정보도 함께 봐야 한다.
- 선택 A: 다른 출처/반대 의견도 찾아보고, 왜 다른 추천이 나오는지 토론한다.
- 선택 B: 추천만 계속 보고 “많이 뜨면 맞다”고 믿는다.
- 확인 질문: 왜 내 화면에는 이것만 뜰까? 다른 근거는 어디서 찾을까?
- 관련 윤리기준: 데이터 관리, 연대성, 안전성

========================
[사례04] 편향: 특정 집단을 고정관념으로 그림
- 주제: 편향 / 연대성 / 침해 금지
- 상황: 포스터 그림에서 어떤 직업/집단이 한 가지 모습으로만 그려졌다. 누군가는 불편하다고 말한다.
- 쟁점: 고정관념은 상처가 될 수 있다. 다양한 모습을 반영해야 한다.
- 선택 A: 다양한 예시를 넣고, 표현을 더 공정하게 수정한다.
- 선택 B: “그냥 농담”이라며 그대로 둔다.
- 확인 질문: 이 표현을 당사자가 보면 어떤 기분일까? 대체 표현은?
- 관련 윤리기준: 연대성, 침해 금지, 안전성

========================
[사례05] 딥페이크/합성: 장난이 커질 수 있음
- 주제: 합성 이미지/영상 / 안전성 / 침해 금지
- 상황: 친구 얼굴을 합성해 재미로 공유하려고 한다. 일부는 싫어할 수 있다.
- 쟁점: 장난이 피해가 될 수 있다. 동의와 중단 기준이 필요하다.
- 선택 A: 당사자 동의 없이는 만들지 않고, 안전한 대체 활동을 한다.
- 선택 B: “친구니까 괜찮다”며 몰래 만들고 공유한다.
- 확인 질문: 동의는 어떻게 받을까? 공유하면 어디까지 퍼질까?
- 관련 윤리기준: 안전성, 침해 금지, 프라이버시 보호
