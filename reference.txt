[국가 인공지능 윤리기준(수업용 요약)]
- 프라이버시 보호: 개인 정보(이름/사진/연락처)를 함부로 쓰지 않기.
- 연대성: 모두가 불편하지 않게(차별/놀림/배제 줄이기).
- 데이터 관리: 자료를 안전하게 보관하고, 목적에 맞게 쓰기(필요한 만큼만).
- 침해 금지: 남의 권리(저작권/초상권 등)를 해치지 않기.
- 안전성: 위험한 행동을 만들거나 퍼뜨리지 않기(사고/유해 정보 주의).

[저작권(초등 고학년 쉬운 설명)]
- 저작권은 ‘작품을 만든 사람의 권리’야.
- 남이 만든 그림/글/사진을 그대로 쓰면 문제가 될 수 있어.
- 안전한 방법:
  1) 허락받기(가능하면 먼저 물어보기)
  2) 출처 밝히기(누가 만들었는지 말하기)
  3) 내 창작 부분과 도움받은 부분을 구분하기(AI 도움도 포함)
  4) 공개 범위 확인하기(반 안에서만? 학교? 온라인?)
- AI 이미지/글도 ‘어디서 배웠는지’가 중요해.
- 너무 비슷하게 따라 만들면 ‘흉내(모방)’가 될 수 있어.

[AI 이미지 생성 수업 주의점(텍스트 없는 그림)]
- 글자/숫자/로고가 들어가지 않게 요청하기.
- 유명 캐릭터/상표/브랜드 느낌이 나지 않게 하기.
- 특정 사람의 얼굴(실제 인물 사진 느낌)은 피하기.
- 누군가를 놀리거나 차별하는 표현은 피하기.
- 공유 목적과 범위를 먼저 정하기.

[교사용 피드백 기준 예시]
- 학생이 ‘허락/출처/목적/공개범위’를 말하면 가점.
- “약관/학교 규칙을 확인해야 한다”를 말하면 가점.
- 대안(바꿔서 만들기/다른 자료 사용/공유 범위 줄이기)을 제시하면 가점.

[사례 분석(딜레마 토론용) - 사례01~사례05]
- 각 사례는 “상황 요약 / 딜레마 / 선택 A/B / 생각 포인트”로 구성됨.
- 토론에서는 A/B를 고르고, 근거/반대 의견/대안을 말하는 연습을 함.

[사례01] 인공지능 화가 ‘브러시’와 비밀의 그림 (저작권)
상황 요약:
- 학생이 AI가 그려준 그림을 자기 작품처럼 제출해 칭찬을 받음.
- 나중에 유명 작가의 원본 그림과 매우 비슷하다는 사실을 알게 됨.
딜레마:
- 편리함 vs 남의 노력(권리) 존중
선택 A:
- 제출/공유 전에 “AI 도움 표시 + 출처/허락 확인 + 비슷함 줄이기”를 한다.
선택 B:
- “어차피 AI가 만든 거니까 괜찮다” 생각하고 그대로 사용한다.
생각 포인트:
- AI가 무엇을 보고 배웠는지(데이터 관리)
- 남의 작품을 닮게 만들면 침해가 될 수 있는지(침해 금지)
- 공개 범위(반/학교/온라인)에 따라 위험이 커지는지(안전성)

[사례02] 발표 자료에 친구 사진과 이름(프라이버시)
상황 요약:
- 발표 자료에 친구 사진/이름을 넣으면 이해가 쉽지만, 온라인 공개 가능성이 있음.
딜레마:
- 이해 쉬움 vs 개인정보 보호
선택 A:
- 얼굴/이름을 가리고(익명), 허락을 받고, 공개 범위를 제한한다.
선택 B:
- “친구니까 괜찮다”며 그대로 넣고 공유한다.
생각 포인트:
- 허락을 받는 기준
- 공개 범위가 넓어질수록 생길 수 있는 문제

[사례03] AI 추천이 한쪽만 보여줌(연대성/공정)
상황 요약:
- AI 추천이 특정 취향/사람에게만 유리하게 보이는 느낌이 든다.
딜레마:
- 편한 추천 vs 공정하게 다양한 선택
선택 A:
- 다른 관점도 찾고, 추천 기준을 확인하며, 불편한 점을 말한다.
선택 B:
- 그냥 편하니까 추천만 따른다.
생각 포인트:
- 다양한 사람에게 공평한지
- 불편한 사람이 생기지 않는지(연대성)

[사례04] 학급 설문 데이터 관리(데이터 관리)
상황 요약:
- 학급 설문 결과에 민감한 내용이 포함됨(연락처/집 정보 등).
딜레마:
- 빠른 공유 vs 안전한 보관
선택 A:
- 필요한 정보만 남기고(최소화), 비공개로 보관한다.
선택 B:
- 모두 편하라고 단체 채팅방에 그대로 올린다.
생각 포인트:
- “필요한 만큼만” 수집/공유하기
- 보관 장소/권한 관리

[사례05] 위험한 방법을 알려달라는 질문(안전성)
상황 요약:
- 누군가 AI에게 위험한 행동을 묻는다(따라 하면 다칠 수 있음).
딜레마:
- 호기심 vs 안전
선택 A:
- 위험한 정보는 거절하고 안전한 대안을 말한다(도움 요청/예방).
선택 B:
- 방법을 자세히 알려준다.
생각 포인트:
- 안전한 대안 제시
- 실제 피해 가능성 줄이기

[수업용 한줄 규칙(학생용)]
- “허락 → 출처 → 목적 → 공개범위”를 먼저 생각하기.
- AI가 도와준 부분은 솔직하게 말하기.
- 남의 작품과 너무 비슷하면 고치기.
- 개인정보는 최소로, 안전하게.
